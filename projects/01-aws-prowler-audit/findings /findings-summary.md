## ⚙️ Setup & Commands Used

### Step 1 — Verify Prerequisites

```bash
# Check Python is installed (Prowler requires Python 3.9+)
python --version

# Check AWS CLI is installed
aws --version
```

### Step 2 — Configure AWS Credentials

```bash
aws configure
```

You will be prompted to enter:

```
AWS Access Key ID:     <your-access-key-id>
AWS Secret Access Key: <your-secret-access-key>
Default region name:   us-east-1          # or your preferred region
Default output format: json
```

> ⚠️ **Never commit your Access Key or Secret Key to GitHub.** These credentials are entered only in your local terminal and stored in `~/.aws/credentials`.

### Step 3 — Install Prowler

```bash
pip install prowler
```

### Step 4 — Verify Prowler Installation

```bash
prowler --version
```

### Step 5 — Run the Audit

Since I was using **AWS**, I ran:

```bash
prowler aws --services iam cloudtrail s3
```

> 💡 **Using a different cloud provider?** Replace `aws` with your provider:
> ```bash
> prowler azure --services defender entra
> prowler gcp --services iam cloudresourcemanager
> ```
> Prowler supports **AWS**, **Azure**, and **GCP** out of the box.

Prowler automatically generates output files in HTML, JSON, JSON-OCSF, and CSV formats on completion.

## Critical Findings (1)

### 🔴 IAM — Active Root Account Access Key

| Field | Detail |
|-------|--------|
| **Service** | IAM |
| **Severity** | Critical |
| **Finding** | Root account has an active access key |
| **Risk** | Root access keys grant unrestricted access to all AWS services and resources with no permission boundaries. If leaked, full account compromise is possible. |
| **Remediation** | Deleted the root access key immediately via AWS IAM Console → Security Credentials |
| **Status** | ✅ Remediated |
